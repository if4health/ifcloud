Funcionamento da Aplicação

API Routes

1. Rota direta:

Nesta rota é possivel rodar "scripts python" que estejam no servidor diretamente.

Rota = http://localhost:3007/if_cloud/run_script/script_name/:script_name

# - ":script_name" = passar o nome de um script que exista no servidor.

2. Rota utilizando Resource FHIR Patient:

Nesta rota é possivel rodar "scripts python" buscando por "Patient" na API CloudECG.

Rota = http://localhost:3007/if_cloud/run_script/patient_id/:id

# - ":id" = passar um "id" de um "Patient" existente no servidor da API.

3. Rota utilizando Resource FHIR Observation:

Nesta rota é possivel rodar "scripts python" buscando por "Observation" na API CloudECG.

Rota = http://localhost:3007/if_cloud/run_script/observation_id/:id

# - ":id" = passar um "id" de um "Observation" existente no servidor da AP.

4. Upload de arquivos:

Nesta rota é possivel upload de "scripts python" para o servidor IFCloud.

Rota = http://localhost:3007/if_cloud/file_uploader

Selecionar um script em python para fazer upload no servidor, somente arquivos ".py".

5. Rotas utilizando Operations:

Nesta rota é possivel upload de "scripts python" para o servidor IFCloud.

Rota = http://localhost:3007/if_cloud/json_form

JSON padrão:

Selecionar um script em python para fazer o upload no servidor, somente arquivos ".py".
